# Service Companies — TCS, Infosys, Wipro, Cognizant

---

## What They Test

Service company interviews focus more on **fundamentals and LLD** than complex distributed systems.

| Area | Weight | Focus |
|------|--------|-------|
| OOP Concepts | 30% | SOLID, polymorphism, abstraction, encapsulation |
| Design Patterns | 25% | Factory, Singleton, Observer, Strategy |
| LLD | 20% | Parking lot, library management, ATM, etc. |
| HLD (basics) | 10% | Client-server, basic 3-tier architecture |
| Java/Spring | 15% | Collections, multithreading, Spring Boot basics |

---

## Common Questions

### OOP & Java

- Explain SOLID with examples
- Difference between abstract class and interface
- When to use composition over inheritance
- Java Collections: HashMap vs TreeMap vs LinkedHashMap
- Multithreading: synchronized, volatile, Executor framework
- Exception handling best practices

### LLD (Always in Java)

- Design ATM system
- Design library management system
- Design parking lot
- Design snake and ladder
- Design elevator system
- Design logging framework

### HLD (Basic Level)

- Design URL shortener
- Design a basic chat application
- Design a file sharing system
- Explain how a website works end-to-end

## Preparation (4 Weeks)

```
Week 1: Java fundamentals + OOP + Collections
Week 2: SOLID principles + Design patterns (this vault: 11_lld)
Week 3: LLD examples (parking lot, library, ATM)
Week 4: Basic HLD + mock interviews
```

## Key Difference from Product Companies

- Less focus on scale (no "how would you handle 1M users?")
- More focus on clean code, proper OOP, Java knowledge
- Design patterns asked as theory + implementation
- Spring Boot / microservices concepts for experienced roles
